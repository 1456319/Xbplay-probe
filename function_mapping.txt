Xbox ONE Game Streaming App - Functionality Map
===============================================

This document classifies individual files and their likely purpose within the project, excluding standard open-source dependencies (e.g., Google, Facebook, Amazon, AndroidX, SpongyCastle, OkHttp).

### Authentication (`com/studio08/xbgamestream/Authenticate`)
- Authentication activities and clients (`LoginClient.java`, `LoginClientV2.java`, etc.): Responsible for presenting Xbox Live login screens and performing OAuth flows. Interfaces with Microsoft APIs (`account.xbox.com/account/signin`) to obtain the authentication tokens required for SmartGlass and xCloud connection.

### Web View and Tokens (`com/studio08/xbgamestream/Web`)
- WebViews and Web clients (`StreamWebview.java`, `WifiClient.java`, `BasicGeckoViewPrompt.java`): Custom web view clients used within the application to render login flows and handle interactions. Includes custom settings like `captureLogs` or `showLoadingDialog`.

### Controller Input Handling (`com/studio08/xbgamestream/Controller`)
- Local physical controller handlers (`ControllerHandler.java`, `RumbleHelper.java`, `InputDeviceContext.java`, `Vector2d.java`): Interfaces with Android's `InputDevice` APIs (`InputManager.InputDeviceListener`, `View.OnKeyListener`, `View.OnGenericMotionListener`) to read physical gamepad inputs, apply rumble feedback, map them using `Vector2d` math, and convert the inputs into JSON/byte payloads to be sent to the console.

### Media Servers (Casting) (`com/studio08/xbgamestream/Servers`)
- Media Servers (`FileServer.java`, `TvServer.java`): Built-in local servers used to host local files or stream content for casting to the Xbox Media Player.

### User Interface & Architecture (`com/studio08/xbgamestream/ui`)
- ViewModels and UI logic (`ControllerViewModel.java`, `GamestreamViewModel.java`, `HomeViewModel.java`): Android ViewModel architecture managing the state and logic for the primary UI screens, including the home menu, remote control layout, and the main gamestream video surface.

### Helper Utilities (`com/studio08/xbgamestream/Helpers`)
- Support and utility classes (`EncryptClient.java`, `FileHelper.java`, `FirebaseAnalyticsClient.java`): Assists with auxiliary tasks such as basic encryption/decryption of local data, file system operations, and telemetry via Firebase.

### Video Format Conversion (`com/studio08/xbgamestream/Converter`)
- `VideoFormatConverter.java`: Logic potentially used to transcode or convert local media files into a format compatible with the Xbox Media Player before casting.

### SmartGlass Cryptography (`crypto`)
- `SgCrypto.java`: Implements Elliptic Curve Diffie-Hellman (ECDH) key exchange and AES-CBC encryption/decryption required to securely communicate with the Xbox console over the SmartGlass protocol. Uses `org.spongycastle` for ECC operations. Contains logic for shared secrets, certificates, and HMAC keys.

### Network Services and Protocol Bridging (`network`)
- `BindService.java`: An Android Service running in the background acting as the central bridge between the app UI and the SmartGlass (`Xbox`) / Nano (`Nano`) clients. It manages controller sequences, connection state, displays persistent notifications, and keeps the sockets alive. Receives events via `ChannelEvents` and `NanoStreamEvents`.
- `SmartGlass.java`, `Listeners.java`, `EventListener.java`: Network logic and event listener interfaces.

### Nano Gamestream Core (`nano/base`)
- `Nano.java`: Core Nano gamestream client that manages UDP and TCP sockets, handling the initial handshake (initially on specified ports with `sessionId` and `xboxAddress`), and routing gamestream packets to appropriate stream handlers (`ControlStream`, `CoreStream`, `VideoStream`, `ControllerInputStream`).
- `VideoDecoder.java`: Implements Android's `MediaCodec` to decode the H264 video stream received from the Xbox and renders it to a `TextureView`. Reconstructs H.264 frames from fragmented UDP packets and handles SPS/PPS configurations. Includes debug logic to dump raw `.h264` streams to external storage.
- `RTPHeader.java`, `StreamerHeader.java`, `ControlHeader.java`, `h264Frame.java`: Parsers for stream headers and video frame reconstruction logic.

### Nano Gamestream Channels (`nano/streams`)
- `CoreStream.java`, `VideoStream.java`, `ControlStream.java`, `ControllerInputStream.java`: Handlers for the distinct multiplexed streams within the Gamestream session, isolating control packets from video or input packets.

### Nano Gamestream Packets (`nano/packets`)
- Nano gamestream packet definitions (`send/`, `response/`): Classes for handshakes, control protocol messages, and formatting controller input frames (e.g., `InputFrame`, `VideoClientHandshake`, `VideoDataResponse`, `ChannelControlResponse`).

### Game Controller State (`nano`)
- `GameController.java`, `GameControllerButtonModel.java`: Maintains the logical state of a connected controller (button presses, analog stick positions) and interfaces with the `Nano` client to package and transmit these state changes.

### SmartGlass Client Core (`xbox`)
- `Xbox.java`: Main SmartGlass client responsible for discovering the console via UDP broadcast, initiating connections, opening channels, sending system inputs, handling power commands (Wake/PowerOff), and listening for packet responses. Integrates tightly with `SgCrypto` for secure communication.

### SmartGlass Packets (`packet`)
- SmartGlass packet definitions (`send/`, `receive/`, `MessagePacket.java`, `SimplePacket.java`): Classes dedicated to structuring, serializing, and deserializing the binary packets used in the SmartGlass protocol (Discovery, Connection, Gamestream Start/Stop, Acknowledgment). Maps complex messages into `SimplePacket` or `MessagePacket` formats.

### Protocol Constants and Mappings (`constants`)
- Protocol constants (`PacketProtocol`, `ChannelTypes`, `Statuses`, `SystemInputButtonMappings`): Fixed byte sequences, status codes, and mappings translating human-readable requests into the specific binary formats expected by the console.

### Utility Functions (`util`)
- `Util.java`: Utility functions for byte array manipulation, UUID processing, endianness conversions (e.g., `byteArrayToIntLE`), and hexadecimal string formatting critical for protocol implementation.
